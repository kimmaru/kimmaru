 김포구원의감격교회 웹사이트 PRD (Product Requirements Document)

 1. 목적과 비전
 - 목적: “구원의 감격”이라는 정체성을 중심으로, 교회 소개, 예배 안내, 설교/미디어 콘텐츠 소비, 오시는 길 안내를 한 번에 해결하는 정갈하고 은혜로운 온라인 허브 구축
 - 핵심 가치: 단순함, 명료함, 감성적 메시지 전달(고급 언어 표현), 모바일 최적의 가독성과 몰입감
 - KPI(정량): 페이지 이탈률 40% 이하, 평균 체류시간 2분 이상, 설교/영상 클릭률 30% 이상, 모바일 속도 점수 80+ (Lighthouse)

 2. 대상 사용자 및 시나리오(Persona)
 - 기존 성도: 예배 시간 확인, 설교 다시보기, 공지/기도제목 파악
 - 새가족/방문자: 교회 철학/분위기 이해, 오시는 길/주차 확인, 첫 방문 준비
 - 온라인 시청자: 최근 예배 실황/영상 시청, 쇼츠 기반 짧은 메시지 소비

 3. 정보 구조(IA) 및 라우팅
 - 글로벌 네비게이션(고정 헤더, 드롭다운)
   - 홈: /
   - 교회소개(드롭다운): /about, /about/pastor(담임목사 인사말), /about/service(예배시간), /about/location(오시는 길)
   - 말씀: /sermon
   - 미디어: /media
   - 404: /* (NotFound)

 4. 핵심 페이지별 요구사항(콘텐츠/UX)
 4.1 홈(/)
 - 히어로: 배경 이미지(main_background.jpeg) 오버레이 + 고급 문장형 서브타이틀(“구원의 감격이 식어버린 시대에… 다시 복음 앞에 섭니다.”)
 - 하이라이트 카드 3종: 담임목사 인사말, 말씀, 예배 안내로 심미적으로 유도(호버 시 깊이감, 스케일)
 - 주일 예배 실황: 최신 YouTube 영상 임베드(실패 시 더미 데이터), 감성 카피와 함께 상단 고정 노출
 - 반응형 타이포그래피(Clamp), 애니메이션은 과하지 않게(프레이머 모션)
 - 접근성: 충분한 대비, 키보드 포커스, 이미지 대체텍스트

 4.2 교회소개(/about)
 - 핵심 문장: “우리는 다시 복음 앞에 섭니다”를 강조 텍스트로 노출
 - 교회 철학/정체성: 정갈한 카드형 섹션, 중간 강조문(하이라이트)

 4.3 담임목사 인사말(/about/pastor)
 - 담임목사 타이틀/성함, 정중하면서도 따뜻한 인사말(문단 구분, 강조문/인용스타일)
 - 마지막 서명 블록(이름, 직함)

 4.4 예배시간(/about/service)
 - 주일 1·2부, 온가족예배 카드: 시간/장소/특징(간결한 표) + 설명문
 - 주간 일정: 월–금/수/금/토/주일/다음세대 섹션을 그리드로 구성

 4.5 오시는 길(/about/location)
 - 카카오 지도 임베드(roughmapLoader 사용): 반응형 컨테이너, 둥근 모서리, 그림자
 - 주소 카드(복사 버튼), 교통 안내(자가용/대중교통/공항) 카드형
 - 문의 섹션: 전화/이메일/운영시간/카카오톡 카드

 4.6 말씀(/sermon)
 - 유튜브 채널 핸들 기반 리스트업(최신/유형 필터 탭: 전체/주일/수요/금요)
 - 상단 플레이어에 선택 영상 임베드, 하단 카드 리스트(썸네일/제목/설명/조회/길이/날짜)
 - 실패 시 더미 데이터로 graceful fallback

 4.7 미디어(/media)
 - 탭: 일반 영상 / YouTube 쇼츠
 - 상단 플레이어 + 영상 정보, 하단 그리드(영상/쇼츠) 카드
 - 인스타그램 홍보 섹션(실제 임베드는 추후 스펙 확정 시 적용)

 4.8 404(/*)
 - 간결한 문구와 홈으로 돌아가기 버튼

 5. 네비게이션/헤더/푸터
 - 헤더: 고정, 스크롤 반응 최소화(현재 일정상 단순 유지), 모바일 드롭다운 메뉴/토글
 - 드롭다운: 모바일에서는 탭-투-토글, 데스크탑 hover 진입/포커스 접근성 고려
 - 푸터: 교회명/슬로건/교단 표식, 예배시간, 연락처, 퀵링크(교회소개/설교/미디어/연락처), 저작권

 6. 미디어/연동 사양
 - YouTube 채널: @Joy_of_salvation_kp
 - YouTube Data API: 최근 영상, 쇼츠, 최신 스트림
   - 환경변수: REACT_APP_YOUTUBE_API_KEY (프론트 런타임)
   - 오류/쿼터 초과 시 더미 데이터 제공(현재 유틸에 구현됨)
 - 카카오맵 roughmap: 반응형 래퍼, 로더 스크립트 onload 처리

 7. 시각/브랜딩 지침(요약)
 - 다크 대비의 고급 톤과 명료한 포인트 컬러(—color-accent)
 - 큰 활자·넓은 행간·충분한 여백으로 정갈함과 격조 전달
 - 카드/섹션 간 명확한 레벨 차이(그림자, 경계선, 배경 톤)

 8. 성능/SEO/접근성
 - LCP: 히어로 이미지 preloading/적절한 사이즈, 폰트 디스플레이 swap
 - SEO: 각 페이지 메타 타이틀/설명, 구조화된 제목 계층, 내비 링크 텍스트 의미화
 - 접근성: 포커스 스타일, aria-label, 명도 대비 준수, 키보드 트랩 방지

 9. 반응형 기준
 - 브레이크포인트: 1200/992/768/576(px) 우선
 - 모바일 퍼스트, 터치 타겟 크기와 간격 유지

 10. 환경/배포/설정
 - 배포: Vercel
 - .env: REACT_APP_YOUTUBE_API_KEY(프론트), (필요 시) GA/메타 픽셀 등 추후
 - 공개 자산: /public/logo.png, /public/main_background.jpeg 등

 11. 운영/콘텐츠 업데이트
 - CMS 미도입(현 단계): 텍스트/일정/연락처는 코드 기반 업데이트
 - 설교/미디어: 유튜브 채널 콘텐츠에 자동 의존(유틸이 최신 목록 수집)

 12. 비기능 요구사항
 - 빌드 경고/린트 에러 0, 타입 안정성 유지
 - CLS/레이아웃 점프 최소화, 애니메이션 과도 방지
 - 네트워크 실패 시 사용자 메시지/더미 가드

 13. 분석/로그(옵션)
 - GA4 페이지뷰, 영상 클릭/체류시간 이벤트(후속 스프린트)

 14. 리스크/제약
 - YouTube API 쿼터 소진 시 데이터 조회 실패 → 더미로 대체, 페이지 품질 하락 가능
 - 카카오 roughmap 정책 변경 시 재적용 필요

 15. 오픈 이슈
 - 다국어(한/영) 필요 여부
 - 인스타그램 실제 임베드 vs 링크 유지
 - 교회 소식/공지 모듈 도입 필요성

 16. 로드맵(스프린트 제안)
 - Sprint 1: PRD 기반 태스크 생성, 네비/홈/푸터 정리, 기본 SEO/접근성, YouTube 연동 안정화
 - Sprint 2: 말씀/미디어 고도화(필터 UX·에러 가드), 오시는 길(지도/교통 UX)
 - Sprint 3: 예배시간/주간일정 운영 편의, 공지/소식(선택), 분석 이벤트

 17. 수용 기준(샘플)
 - 홈: 히어로 카피/배경/카드 3종 정상 렌더, 최신 예배 영상 표시(또는 더미), 768px 이하에서도 시각 안정
 - 말씀: 채널 영상 20개 로드(또는 더미), 탭 별 필터 정상, 플레이어 전환 부드러움
 - 미디어: 일반/쇼츠 탭, 선택 영상 상단 플레이, 썸네일/제목/날짜/조회수 노출
 - 오시는 길: 카카오 지도가 로드되며 반응형, 주소 복사 버튼 동작
 - 예배시간: 주일/주간 일정 카드 명확, 모바일 가독성 우수

 부록 A. 현재 구현 요약(코드 반영)
 - 라우트: /, /about, /about/pastor, /about/service, /about/location, /sermon, /media, /*
 - 헤더: 고정 헤더, 드롭다운(모바일 토글), 활성 링크 하이라이트
 - 푸터: 슬로건/교단/예배시간/연락처/퀵링크/저작권
 - YouTube 연동: 채널 핸들(@Joy_of_salvation_kp) 기반 조회, Failure fallback 구현됨
 - 지도: Kakao roughmap 스크립트 onload 후 렌더
 - 스타일: styled-components, GlobalStyles 변수 사용, framer-motion 애니메이션
